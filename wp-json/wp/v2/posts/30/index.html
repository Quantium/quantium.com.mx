Despu\u00e9s de una larga espera al fin implementamos el sistema de control de versiones en el trabajo, actualmente est\u00e1 s\u00f3lo para pruebas y revisar la configuraci\u00f3n. El esquema de trabajo que hemos platicado es bloquear los archivos binarios como .fla y .jpg cuando se est\u00e9n editando y utilizar el esquema de copiar-modificar-fusionar para los .as.<\/p>\n

Estamos utilizando un respositorio Subversion<\/a> y como cliente Tortoise<\/a>.<\/p>\n

Y justo a tiempo, porque con 3 equipos de desarrollo trabajando sobre el mismo proyecto hoy tenemos ya demasiados problemas controlando las versiones y perdemos demasiado tiempo copiando todo el tiempo toda una carpeta con dosmil y pico de archivos s\u00f3lo para probar un peque\u00f1o cambio.<\/p>\n

Pocos proyectos conosco que trabajen con un sistema de control de versiones cuando se trata de actionscript. Uno de ellos es Desktoptwo<\/a> mejor conocido en su versi\u00f3n anterior computadora.de<\/a> desarrollado por sapotek<\/a>, muy bueno proyecto, aunque comercialmente opino lo mismo de Web 2.0<\/a> que Rogelio Bernal en este art\u00edculo<\/a> en Baqu\u00eda. Adem\u00e1s est\u00e1n todos (o casi todos) los frameworks hosteados en osflash<\/a>.<\/p>\n
